# Lucky Exchange Bot ☘️

Профессиональный клон крипто-обменника в Telegram с веб-админкой и динамическим управлением.

```
┌─────────────┐      ┌──────────────┐      ┌─────────────┐
│ Telegram Bot│◀────▶│   SQLite DB  │◀────▶│  Web Admin  │
└─────────────┘      └──────────────┘      └─────────────┘
      │                     │                     │
   aiogram 3           SQLAlchemy             FastAPI
```

## 🚀 Быстрый старт

### 1. Подготовка окружения
Убедитесь, что у вас установлен [uv](https://github.com/astral-sh/uv).

```powershell
# Установка зависимостей
uv sync
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корне проекта (или отредактируйте существующий):

```env
BOT_TOKEN=ваш_токен_от_botfather
ADMIN_ID=ваш_телеграм_id
WEB_ADMIN_USERNAME=admin
WEB_ADMIN_PASSWORD=ваш_пароль
SECRET_KEY=секретный_ключ_для_сессий
```

### 3. Инициализация данных (Курсы и Методы)
Запустите скрипт для наполнения базы актуальными данными и создания необходимых таблиц:

```powershell
uv run python init_data.py
```

### 4. Запуск в режиме разработки (TurboPack)
Чтобы бот автоматически перезагружался при изменении кода:

```powershell
uv run python dev.py
```

## 🛠 Управление проектом

### Веб-Админка
Доступна по адресу: `http://localhost:8000/admin`
- **Заявки**: Просмотр и изменение статусов (Pending -> Completed).
- **Курсы валют**: Динамическое изменение цен покупки/продажи.
- **Методы оплаты**: Добавление новых кнопок (СБП, Карты, QR и т.д.).
- **Настройки (Реквизиты)**: Управление кошельками для приема оплаты.

### Динамические реквизиты
Чтобы изменить кошелек, который бот выдает пользователю при продаже:
1. Зайдите в Админку -> **Настройки (Реквизиты)**.
2. Создайте запись с ключом `wallet_BTC` и вашим адресом в поле `value`.
3. Бот подхватит изменения мгновенно.

## 🏗 Архитектура

- **`bot/`**: Логика Telegram бота на aiogram 3.
  - **`handlers/`**: Модульные обработчики (Обмен, Чистка, Профиль).
  - **`middlewares/`**: Автоматическое управление сессиями БД.
- **`web/`**: Веб-интерфейс на FastAPI + SQLAdmin.
- **`core/`**: Общие модели данных и конфигурация.
- **`assets/`**: Анимации и изображения.

## ⚡ Особенности реализации
- **File ID Caching**: Бот сохраняет идентификаторы загруженных видео в таблицу `file_cache`. Это позволяет отправлять 9МБ анимации мгновенно, не загружая их повторно.
- **Sanitization**: Весь пользовательский ввод экранируется для предотвращения XSS в интерфейсе Telegram.
- **State Machine (FSM)**: Надежное управление многошаговыми процессами обмена.
